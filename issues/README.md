# issues/

Challenge statements and the starter issue backlog for the hackathon.

| File | Purpose |
| --- | --- |
| `challenge_statements.md` | The challenge tracks participants can build against |
| `issue_backlog.md` | Concrete, scoped starter issues with difficulty labels |

## How To Use This Folder

1. Read the challenge statements and pick a track (or define your own — the
   tracks are directions, not limits).
2. Scan the backlog for a starter issue that matches your skills, or propose a
   new one.
3. Announce what you are working on by opening a GitHub issue that references
   the backlog item, so teams do not collide.
4. Build using the synthetic datasets and submit a pull request following
   [CONTRIBUTING.md](../CONTRIBUTING.md).

Maintainers: backlog items can be promoted to GitHub issues using the issue
templates in `.github/ISSUE_TEMPLATE/`.
