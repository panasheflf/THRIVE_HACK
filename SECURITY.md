# Security and Sensitive Data Reporting

This project does not accept real participant-level health data in public
issues, pull requests, examples, tests, screenshots, logs, or documentation.

## Do Not Post Publicly

Do not post:

- Real menstrual health records
- Names, phone numbers, emails, or identity-linked records
- Exact cycle dates
- Free-text health responses copied from real people
- Precise location data
- Consent records
- Partner-submitted raw files
- Credentials, API keys, tokens, or private operational details

## Reporting Sensitive Data Exposure

If you believe sensitive data or a secret has been exposed:

1. Do not create a public issue containing the sensitive content.
2. Contact the repository maintainers privately through GitHub.
3. If private reporting is not available, open a public issue that says only
   that a sensitive-data review is needed. Do not include the data, link, secret,
   screenshot, or identifying details.

## Maintainer Response

Maintainers should:

1. Remove or rotate exposed content as appropriate.
2. Preserve only the minimum audit details needed for remediation.
3. Review whether documentation, validation, or contribution guidance needs to
   be improved.
4. Avoid re-sharing sensitive content while resolving the report.
