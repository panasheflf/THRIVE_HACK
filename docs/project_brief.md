# Project Brief

## Overview

Menstrual Health Open is an open-source infrastructure initiative for ethical
menstrual health research.

Menstrual health affects people globally, but research is often fragmented by
inconsistent data collection, limited interoperability, weak tooling, and the
sensitive nature of reproductive health data. This project creates shared tools,
standards, workflows, and governance resources that make menstrual health data
easier to collect, validate, analyze, compare, and use responsibly.

## What This Is

This is an open-source infrastructure project. The public repository contains:

- Data schemas and data dictionaries
- Synthetic menstrual health datasets
- Validation and quality-checking tools
- Lightweight validation baseline
- Open-ended project inspirations
- Governance and ethics resources
- Contribution guidelines and challenge tracks

## What This Is Not

This is not an open raw-data project.

The public repository does not contain:

- Real participant-level records
- Exact cycle dates
- Raw symptoms, diagnoses, pregnancy history, contraceptive history, or related
  private health details
- Raw free-text responses
- Precise location data
- Partner-submitted raw files
- Consent records
- Identity-linked records
- Rare-condition or small-community data
- Partner contracts or private agreements
- Credentials, secrets, or internal operational systems

## Hackathon Goal

The hackathon focuses on building open tools, standards, workflows, and
governance resources for ethical menstrual health research. Participants can use
the starter repository as a shared foundation while defining their own direction.

All hackathon work should use synthetic datasets, sample datasets, or approved
aggregate assumptions. Real participant-level data should not be used in the
hackathon environment.

The project inspirations are examples, not limits. Participants may contribute
code, documentation, governance, research methods, translation, accessibility
review, community engagement materials, analysis patterns, or other reusable
infrastructure.

## Long-Term Goal

After the hackathon, the repository should remain useful as an open-source
ecosystem for:

- Researchers designing interoperable menstrual health studies
- Developers building validation, privacy, and analysis tools
- Community organizations adapting forms and materials
- Public health teams creating dashboards and aggregate reporting workflows
- Educators creating accessible menstrual health research resources
- Contributors improving governance, localization, and accessibility

## Public Positioning

This project is an open-source infrastructure initiative for ethical menstrual
health research. The code, schemas, documentation, synthetic datasets, research
methods, and governance resources are open for public contribution. Sensitive
participant-level data collected through partners is protected and not publicly
released. Approved researchers may request controlled access under the project's
data governance and ethical review process.

## Core Principle

Open the infrastructure. Protect the people.
